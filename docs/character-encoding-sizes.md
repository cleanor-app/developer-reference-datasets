# How many bytes is a character in UTF-8 vs UTF-16 vs UTF-32?

**In UTF-8 a Latin (ASCII) character is 1 byte, an accented-Latin or Greek/Cyrillic letter is 2, a CJK glyph is 3, and most emoji are 4. UTF-16 is 2 bytes for anything on the Basic Multilingual Plane and 4 for the rest. UTF-32 is always 4 bytes per code point.** Measured per script in [`data/encoding-sizes.csv`](../data/encoding-sizes.csv).

| Latin-1 accented | Café, naïve, résumé | 27 | 32 | 1.19 |
| Greek | Καλημέρα κόσμε | 14 | 27 | 1.93 |
| Cyrillic | Здравствуй, мир | 15 | 28 | 1.87 |
| Arabic | مرحبا بالعالم | 13 | 25 | 1.92 |
| Devanagari | नमस्ते दुनिया | 13 | 37 | 2.85 |
| Chinese (Han) | 你好，世界 | 5 | 15 | 3.00 |
| Korean (Hangul) | 안녕하세요 세계 | 8 | 22 | 2.75 |
| Emoji | 😀🎉🚀 | 3 | 12 | 4.00 |

| Emoji (flag) | 🇺🇸 | 2 | 8 | 4.00 |

## Why it matters

- **A database column of "255 characters" is not 255 bytes.** In UTF-8 the same 255 CJK characters are 765 bytes; 255 emoji can be over 1000. Size columns by bytes, not characters.
- **A single emoji is not one character.** A flag is 2 code points (regional indicators) and 8 UTF-8 bytes; a family emoji is a ZWJ sequence of 7 code points and 25 bytes. `"👨‍👩‍👧‍👦".length` is 11 in JavaScript (UTF-16 code units), not 1.
- **This is why QR "byte mode" payloads and text storage differ by language:** the byte count, not the character count, is what fills the capacity.

## Method

`utf8 = Buffer.byteLength(s, 'utf8')`; UTF-16 bytes `= s.length × 2` (JavaScript strings are UTF-16); UTF-32 bytes `= [...s].length × 4` (spread iterates by code point). Generated by [`generators/encoding-sizes.mjs`](../generators/encoding-sizes.mjs). Add your own strings and re-run.
